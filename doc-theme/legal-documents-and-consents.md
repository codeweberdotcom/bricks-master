# Юридические документы и согласия на обработку ПДн

Модуль объединяет хранение юридических документов (тип записи **Legal**) и настройку согласий на обработку персональных данных (ПДн): привязка документов к формам, рассылкам и отображение на сайте.

## 1. Назначение

- **Единое место для юридических текстов** — политика конфиденциальности, оферта, правила сайта, согласие на обработку ПДн и т.п. хранятся в виде записей типа «Legal».
- **Связь документов с формами и рассылкой** — в формах и подписке используются чекбоксы согласия со ссылками на выбранные документы; факт согласия пользователя фиксируется.
- **Соответствие требованиям к ПДн** — возможность обосновать, какой документ был показан пользователю и на какую версию он дал согласие.
- **Куки-баннер** — уведомление о использовании cookie и кнопка «Согласен» с записью согласия в cookie и ссылкой на политику cookie.
- **Интеграция с инструментом персональных данных WordPress** — экспорт и удаление данных по запросу субъекта ПДн через стандартный интерфейс WP (Инструменты → Экспорт/Удаление персональных данных), с гибким учётом согласий и отправок форм.

## 3. Тип записи Legal

- **Тип записи:** `legal` (в интерфейсе — «Legal»).
- **Поддержка:** заголовок, редактор, краткое описание, ревизии, комментарии, автор.
- **На сайте:** записи доступны по постоянным ссылкам, может быть архив (список всех юридических документов).
- В настройках для каждого типа документа можно **скрыть** выбранную страницу из архива (чекбокс «Hide from archive»).

Документы можно создавать вручную (Legal → Add Legal) или через страницу **Legal Settings** кнопкой «Create» для предустановленных типов.

## 4. Настройки юридических документов (Legal Settings)

На странице **Legal → Legal Settings** для каждого типа документа задаётся:

1. **Выбор страницы** — из выпадающего списка существующих записей типа `legal`. Выбранная запись считается «официальной» страницей этого типа (например, политика конфиденциальности).
2. **Кнопка «Create»** — создаёт новую запись типа `legal` по шаблону (заголовок, slug, начальный контент из кода темы) и сразу подставляет её в настройку.
3. **Скрыть из архива** — чекбокс: не показывать эту запись в общем архиве юридических документов.
4. **Шорткод** — для каждого типа выводится шорткод вида `[url_название]`, например `[url_privacy-policy]`. На любой странице или в виджете он подставит URL выбранного документа.

Предустановленные типы документов (задаются в коде темы, список может расширяться):

- Политика конфиденциальности (Privacy Policy)  
- Пользовательское соглашение (Terms of Use)  
- Политика использования cookie (Cookie Policy)  
- Согласие на обработку данных (Consent to Data Processing)  
- Договор публичной оферты (Public Offer Agreement)  
- Правила использования сайта (Website Usage Rules)  
- Условия доставки (Delivery Terms)  
- Условия возврата (Return Policy)  
- Согласие на рассылку (Email Marketing Consent)  
- Лицензионное соглашение (License Agreement)  
- Информация о продавце (Seller Information)  

Для политики конфиденциальности при создании страницы через «Create» её ID дополнительно сохраняется в настройке WordPress «Страница политики конфиденциальности» (`wp_page_for_privacy_policy`).

## 5. Согласия на обработку ПДн

### Интеграция с Codeweber Form

В редакторе формы (тип записи **Codeweber Form**) есть блок **«Согласия» (Consents)**:

- Добавляются чекбоксы согласия; каждый привязывается к одному документу типа `legal` (или к странице политики конфиденциальности).
- В подписи чекбокса можно использовать плейсхолдеры: `{document_url}`, `{document_title}`, `{document_title_url}`, `{form_id}` — подставляются ссылка на документ, его название, название как ссылка, ID формы.
- При отправке формы фиксируется, что пользователь дал согласие на указанный документ (и при необходимости — версию документа). Данные сохраняются в таблице отправок Codeweber Forms и, при наличии авторизованного пользователя, в реестре согласий.

Чекбоксы выводятся с именами вида `form_consents_<ID_документа>`. Обработка и сохранение согласий встроены в API форм Codeweber.

### Интеграция с Contact Form 7 (CF7)

Тема интегрирует CF7 с той же системой юридических документов и учёта согласий:

1. **Сохранение отправок** — при успешной отправке формы CF7 (`wpcf7_mail_sent`) данные и приложенные файлы сохраняются в базу Codeweber Forms (та же структура, что и у нативных форм). Таким образом, все отправки (и CF7, и Codeweber Form) оказываются в одном месте.

2. **Поля согласия в CF7** — в форме CF7 можно добавить чекбоксы согласия, которые автоматически распознаются и сохраняются как привязка к документу Legal. Поддерживаются форматы имён полей:
   - **`form_consents_<ID>`** — ID документа типа `legal` (или страницы политики конфиденциальности). Пример: `form_consents_42`.
   - **`soglasie-<slug>`** — slug записи Legal (например, `soglasie-privacy-policy`, `soglasie-consent-processing`). Документ ищется по `post_name` в CPT `legal`.
   - **`form_consents[ID]`** — устаревший формат, сохраняется для совместимости.

   Если пользователь отметил чекбокс, в отправку записывается соответствие «документ — согласие дано».

3. **Ссылки на документы в тексте CF7** — в тексте письма, сообщения об успехе или в подписи к полям можно использовать шорткоды CF7, которые подставляют URL документа. Обрабатываются (в т.ч. через общий Consent Helper темы):
   - **`[cf7_privacy_policy]`** — ссылка на страницу политики конфиденциальности.
   - **`[cf7_legal_consent_link id="…"]`** — ссылка на выбранный юридический документ.
   - **`[cf7_mailing_consent_link id="…"]`** — ссылка на документ согласия на рассылку.

   Таким образом, в письме или на форме можно вывести «Я согласен с [политикой конфиденциальности]» со рабочей ссылкой на актуальный документ из Legal.

Итог: формы CF7 и Codeweber Form используют общий набор документов (Legal + политика конфиденциальности), единый формат полей согласия и единое хранилище отправок и согласий.

### Связь с формами (общее)

Так обеспечивается явное согласие на обработку ПДн в соответствии с выбранным юридическим документом — и в нативных формах Codeweber Form, и в формах Contact Form 7.

### Согласие на рассылку

В настройках рассылки (newsletter) можно указать документ для согласия на обработку email (опция, привязанная к типу документа «Email Marketing Consent» в Legal Settings). Подписчик связывается с этим документом и версией при даче согласия.

### Хранение и версии

- Согласия пользователей сохраняются в связке с идентификатором документа (и при необходимости с версией/датой документа).
- Для записей типа `legal` используются ревизии WordPress — можно определить, на какую версию документа было дано согласие (в т.ч. для проверок и обращений по ПДн).

## 6. Инструмент персональных данных WordPress

Тема интегрирована с **инструментом экспорта и удаления персональных данных** WordPress (**Инструменты → Экспорт персональных данных / Удаление персональных данных**). Это даёт возможность гибко управлять согласиями и исполнять запросы субъектов ПДн в одном месте.

### Как это устроено

- В теме подключён модуль **Personal Data V2**: менеджер провайдеров регистрирует экспортеры и эрасеры через стандартные хуки WordPress `wp_privacy_personal_data_exporters` и `wp_privacy_personal_data_erasers`.
- При запросе **экспорта** или **удаления** данных по email администратор запускает процесс из админки (Инструменты → персональные данные). WordPress по очереди вызывает зарегистрированные провайдеры; каждый провайдер возвращает или экспортирует свои данные по этому email, либо удаляет их.

### Зарегистрированные провайдеры

- **Codeweber Forms** — отправки форм (таблица `wp_codeweber_forms_submissions`), приложенные файлы; экспорт и удаление по email.
- **Contact Form 7** — отправки CF7, сохранённые в системе темы; экспорт и удаление по email.
- **Согласия (Consent)** — записи о выданных пользователем согласиях на документы (в т.ч. привязка к документу и версии); экспорт и удаление по email.
- **Подписка на рассылку (Newsletter)** — подписчики, согласия на рассылку; экспорт и удаление по email.
- **Отзывы (Testimonials)** — отправки форм отзывов; экспорт и удаление по email.

Таким образом, через один интерфейс WordPress можно выполнить экспорт всех данных человека по email (включая согласия и отправки форм) или удалить их при запросе на забвение, что упрощает соблюдение 152-ФЗ и GDPR.

## 7. Шорткоды

### Ссылки на юридические документы

- **`[url_ slug ]`** — подставляет URL выбранного в Legal Settings документа. Пример: `[url_privacy-policy]`, `[url_consent-processing]`.
- **`[link_ slug ]`** — выводит ссылку на документ; атрибуты задают текст ссылки (например, `type_link="title"` — подставить заголовок документа). Удобно для формулировок вида «Я согласен с [документом]».

Slug берётся из настроек типа документа (например, `privacy-policy`, `terms-of-use`, `consent-processing`).

### Переменные из настроек темы (Redux)

В тексте юридических документов (и в любом контенте, где обрабатываются шорткоды WordPress) можно подставлять значения из **настроек Redux** темы с помощью шорткода **`[redux_option]`**. Это позволяет один раз задать реквизиты, контакты и формулировки в настройках темы и использовать их во всех документах без дублирования.

**Синтаксис:** `[redux_option key="ключ_поля" default="значение по умолчанию" format="формат" list="inline|ul"]`

- **`key`** — обязательный; ключ поля в Redux (как в разделе настроек темы, посвящённом юридическим данным и ПДн, так и в других разделах — компания, банк, адреса и т.д.).
- **`default`** — текст, который выводится, если значение в настройках пустое.
- **`format`** — для полей с датой: формат вывода (например, `d.m.Y` для даты в виде 31.12.2025).
- **`list`** — для полей-массивов (например, чекбоксы): `inline` — через запятую в одну строку, `ul` — маркированный список.

**Примеры для юридических документов (ключи из раздела Redux «Юридические данные / ПДн»):**

- `[redux_option key="legal_entity"]` — полное наименование оператора (ИП/ООО).
- `[redux_option key="legal_entity_short"]` — краткое наименование.
- `[redux_option key="legal_ogrnip"]`, `[redux_option key="legal_kpp"]`, `[redux_option key="taxpayer_identification_number"]` — ОГРНИП, КПП, ИНН.
- `[redux_option key="legal_ogrnip_date" format="d.m.Y"]` — дата госрегистрации.
- `[redux_option key="responsible_person"]`, `[redux_option key="phone_responsible_person"]`, `[redux_option key="email_responsible_person"]` — ответственное лицо и контакты.
- `[redux_option key="storage_address"]` — адрес хранения персональных данных.

Аналогично можно использовать ключи из разделов «Реквизиты компании», «Адреса», «Банк» (например, `juri-city`, `fact-street`, `bank-name`) и другие. В описаниях полей в Redux часто указан пример шорткода для этого поля (например: `[redux_option key="legal_entity"]`).

## 8. Куки-баннер

В теме есть встроенный баннер согласия на использование cookie. Он показывается внизу экрана (модальное окно Bootstrap) при первом заходе пользователя и скрывается после нажатия «Согласен».

### Где настраивается

Настройки находятся в **Redux Framework** (в теме: раздел настроек с юридическими данными и ПДн):

- **Cookie Data** (подраздел):
  - **Enable Cookie Banner** — включить/выключить баннер.
  - **Cookie Expiration Date** — срок хранения cookie согласия в днях (по умолчанию 365).
  - **Welcome Text Cookie Banner** — текст баннера (визуальный редактор). Поддерживаются шорткоды; в тексте по умолчанию используется ссылка на политику cookie через `[url_cookie-policy]`.

Текст по умолчанию сообщает о использовании cookie для работы сайта, сохранения настроек и аналитики (в т.ч. Яндекс.Метрика) и содержит ссылку на документ «Политика использования cookie» из Legal Settings.

### Как работает

- Баннер выводится в `wp_footer`, только если включён в настройках и у пользователя ещё нет сохранённого согласия (cookie с именем вида `user_cookie_consent_<hash_домена>`).
- Не показывается поисковым роботам (по User-Agent) и не показывается на самой странице политики cookie (чтобы не мешать чтению документа).
- По нажатию «Согласен» в cookie записываются: дата/время согласия, страница, с которой дано согласие, и источник перехода (referrer). Срок действия cookie задаётся в настройках (в днях).

Так фиксируется факт и момент согласия на использование cookie в соответствии с политикой cookie сайта.

### Cookie Scanner (сканирование cookie)

В том же разделе Redux есть подраздел **Cookie Scanner**: кнопка «Scan Frontend Cookies» открывает окно с фронтендом сайта, собирает список cookie браузера и выводит их в админке в виде таблицы (идентификатор, значение, владелец, срок хранения, тип). Назначение по имени подставляется из **библиотеки** (только чтение, правка вручную): при русском языке админки — `cookies-known-ru.json`, иначе — `cookies-known.json`. Сохранение из сканера пишет в файл по языку: `cookies-found-ru.json` или `cookies-found.json` (мерж при каждом сохранении). Так в политику cookie можно собирать перечень реально используемых cookie отдельно для каждого языка.

Шорткод **`[codeweber_cookie_table]`** выводит таблицу с этими данными (идентификатор, назначение, срок хранения, тип) — его можно вставить в текст документа «Политика cookie», чтобы перечень cookie подставлялся автоматически из тех же JSON-файлов по текущему языку.

## 9. Юридическая польза

- **Прозрачность** — пользователь видит актуальный текст документа по ссылке из формы или подписки.
- **Фиксация согласия** — сохраняется привязка «пользователь — документ (и версия)», что помогает при запросах Роскомнадзора и обращении субъекта ПДн.
- **Единый реестр** — все основные юридические тексты и согласия на обработку ПДн собраны в Legal и в настройках Legal Settings, что упрощает обновление и аудит.

Ответственность за содержание документов и соответствие 152-ФЗ и иным нормам остаётся на владельце сайта; модуль предоставляет инструменты для хранения документов и учёта согласий.
