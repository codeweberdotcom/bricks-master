Обнови версию темы CodeWeber и добавь в CHANGELOG.md записи из коммитов, которые ещё не внесены.

Рабочая директория темы: `wp-content/themes/codeweber/`

---

# ФАЗА 1: Сбор информации

## Шаг 1: Определить текущую версию

Прочитай `style.css` и найди текущую версию в заголовке:
- `Version: X.Y.Z`

## Шаг 2: Определить новую версию

Из аргумента `$ARGUMENTS` извлеки новую версию (например `1.1.0`).
Если версия не указана — спроси пользователя, предложив варианты:
- **patch** (X.Y.Z+1) — баг-фиксы
- **minor** (X.Y+1.0) — новый функционал
- **major** (X+1.0.0) — breaking changes

Если указан флаг `--dry-run` — только показать что будет изменено, ничего не менять.

## Шаг 3: Найти незадокументированные коммиты

Проверь, существует ли `CHANGELOG.md` в корне темы.

**Если CHANGELOG.md существует:**
Прочитай его и определи дату/версию последней записи. Затем выполни:

```bash
cd wp-content/themes/codeweber
git log --oneline --after="<дата последней записи>" -- .
```

Сравни коммиты с тем, что уже задокументировано, чтобы найти незадокументированные.

**Если CHANGELOG.md не существует:**
Получи все коммиты (или разумное количество последних):

```bash
cd wp-content/themes/codeweber
git log --oneline -- . | head -50
```

## Шаг 4: Классифицировать коммиты

Прочитай diff каждого незадокументированного коммита (`git show --stat <hash>` и при необходимости `git show <hash>` для деталей).

Распредели изменения по категориям Keep a Changelog:
- **Added** — новый функционал, новые CPT, новые шаблоны, интеграции
- **Changed** — изменения существующего функционала
- **Fixed** — исправления багов
- **Removed** — удалённый функционал
- **Security** — исправления безопасности
- **Deprecated** — функционал, помеченный как устаревший

Игнорируй:
- Коммиты, которые только меняют `dist/` (пересборка без изменений src)
- Merge-коммиты без содержательных изменений
- Коммиты только с изменениями документации (doc_claude/, CLAUDE.md), если пользователь не попросит иное

## Шаг 5: Показать план

Выведи:

```
## Changelog: <текущая версия> → <новая версия>

### Новые записи:

#### Added
- ...

#### Changed
- ...

#### Fixed
- ...

### Файлы, которые будут изменены:
- style.css (Version в заголовке темы)
- CHANGELOG.md (новая секция / создание файла)
```

## ОБЯЗАТЕЛЬНАЯ ПАУЗА

**НЕ МЕНЯЙ ФАЙЛЫ.** Спроси пользователя:

> Применить эти изменения? Можешь отредактировать записи или изменить версию.

Жди ответа. Если пользователь хочет скорректировать — внеси правки в план и снова покажи.
Переходи к Фазе 2 ТОЛЬКО после явного подтверждения.

# ФАЗА 2: Применение (только после подтверждения)

## Шаг 6: Обновить версию в style.css

Замени `Version: <старая>` на `Version: <новая>` в заголовке темы.

## Шаг 7: Обновить или создать CHANGELOG.md

**Если CHANGELOG.md существует:**
Добавь новую секцию сразу после строки `---` (после описания, перед первой записью).

**Если CHANGELOG.md не существует:**
Создай файл с таким содержимым:

```markdown
# Changelog

Все значимые изменения темы CodeWeber будут задокументированы в этом файле.

---

## [<новая версия>] - <сегодняшняя дата YYYY-MM-DD>

### Added
- ...

### Changed
- ...

### Fixed
- ...

---

## Формат

Этот changelog следует принципам [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).
```

Формат записей:

```markdown
## [<новая версия>] - <сегодняшняя дата YYYY-MM-DD>

### Added
- **Компонент/Область**: описание

### Changed
- **Компонент/Область**: описание

### Fixed
- **Компонент/Область**: описание
```

Не используй эмодзи в заголовках секций.

## Шаг 8: Итоговый отчёт

Выведи:

```
## Готово: версия обновлена <старая> → <новая>

### Изменённые файлы:
- style.css — Version в заголовке
- CHANGELOG.md — добавлена секция [<новая>]

### Следующие шаги:
- Проверь CHANGELOG.md — убедись что записи корректны
- Запусти `npm run build` для пересборки (обновит dist/)
- Закоммить: `release: bump theme version to <новая>`
```

## Шаг 9: Предложить коммит

Спроси:

> Закоммитить обновление версии? (`release: bump theme version to <новая>`)

Если пользователь подтвердит — выполни `/commit`.
