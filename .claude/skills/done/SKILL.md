Выполни финальные шаги для завершения задачи в теме CodeWeber.

## Шаг 1: Сборка

Запусти `npm run build` из директории темы.
Если сборка упала — остановись и сообщи об ошибке. Дальше не продолжай.

## Шаг 2: Определи изменённые файлы

Запусти `git diff --name-only` и `git status`.
Запомни список изменённых PHP-файлов для следующего шага.

## Шаг 3: Проверка безопасности

Для каждого изменённого PHP-файла выполни быструю проверку:
- Нет `$_POST`/`$_GET` без sanitize
- Нет `echo` без esc_*
- Нет `file_put_contents()` / debug-кода

Если найдены проблемы — сообщи и спроси продолжать ли.

## Шаг 4: Проверка i18n

Для каждого изменённого PHP-файла проверь:
- Новые user-facing строки обёрнуты в `__()` / `_e()` с доменом `codeweber`

## Шаг 5: Обновление документации

Определи какие файлы `doc_claude/` нужно обновить по изменённому коду.
Обнови соответствующие документы.

## Шаг 6: Коммит

Составь сообщение коммита на основе:
- Аргумента `$ARGUMENTS` (если передан)
- Анализа изменений

Создай коммит со всеми изменениями включая обновлённую документацию.

## Итоговый отчёт

Выведи краткий отчёт:
- Что сделано
- Что скомпилировано
- Какие документы обновлены
- Hash коммита
