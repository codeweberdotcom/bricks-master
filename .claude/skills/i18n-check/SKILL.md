Проверь файл `$ARGUMENTS` на готовность к переводу в WordPress.

**Текстовый домен темы:** `codeweber`

Проверь по следующим пунктам:

**Строки для перевода:**
- [ ] Все user-facing строки (текст кнопок, сообщения, заголовки) обёрнуты в функции перевода
- [ ] Используются правильные функции:
  - `__( 'string', 'codeweber' )` — для строк без вывода
  - `_e( 'string', 'codeweber' )` — для строк с выводом
  - `_n( 'single', 'plural', $count, 'codeweber' )` — для форм числа
  - `_x( 'string', 'context', 'codeweber' )` — для строк с контекстом
  - `esc_html__()` / `esc_attr__()` — для строк с экранированием

**Ошибки текстового домена:**
- [ ] Нет строк с неправильным доменом (не `codeweber`)
- [ ] Нет захардкоженных строк вне функций перевода (особенно в атрибутах HTML, JS-переменных)

**JS-строки:**
- [ ] Строки для JS переданы через `wp_localize_script()` с переводом на PHP-стороне

**Исключения** (не нужно переводить):
- Технические строки: slug, CSS-классы, meta-ключи, имена хуков
- Строки только для разработчиков (комментарии, debug)

Выведи:
1. Список найденных проблем с номерами строк
2. Строки без перевода (если есть) — конкретный текст и строка
3. Итог: готов к переводу / требует доработки
