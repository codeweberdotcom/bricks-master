Реализуй новый функционал в теме CodeWeber: `$ARGUMENTS`

---

## Шаг 1: Коммит текущего состояния

Запусти `git status`.

- Если есть незакоммиченные изменения — создай коммит перед началом работы.
- Если рабочее дерево чистое — продолжай.

---

## Шаг 2: Анализ

Определи что именно нужно реализовать:

- Прочитай `./style.css` — определи `PARENT_SLUG` (строка `Template:`).
- Прочитай `CLAUDE.md` текущей темы.
- По типу задачи прочитай нужный документ из `../PARENT_SLUG/doc_claude/`:

| Тип задачи | Документ |
|------------|----------|
| Новый CPT | `cpt/CPT_HOW_TO_ADD.md` |
| Шаблон / переопределение | `templates/ARCHIVE_SINGLE_PATTERNS.md` |
| AJAX / fetch | `api/AJAX_FETCH_SYSTEM.md` |
| Настройки Redux | `settings/REDUX_OPTIONS.md` |
| Формы CF7 | `forms/CF7_INTEGRATION.md` |
| Общая архитектура | `architecture/THEME_OVERVIEW.md` |

- Проверь, не реализован ли уже похожий функционал в parent или child теме.

---

## Шаг 3: План

Составь и покажи пользователю план реализации:

- Какие файлы будут созданы / изменены
- В parent или child теме (по умолчанию — всё новое идёт в child)
- Возможные риски или зависимости

**Дождись подтверждения пользователя перед реализацией.**

---

## Шаг 4: Реализация

Реализуй по согласованному плану.

Правила:
- Новый функционал — только в child теме, не трогай parent
- Безопасность: `$_POST`/`$_GET` — только через `sanitize_*`, вывод — через `esc_*`
- Строки — оборачивай в `__()` / `_e()` с text domain из `style.css`
- Frontend — только Bootstrap-классы, без кастомных стилей для того что покрыто темой

---

## Шаг 5: Отчёт

Выведи краткий отчёт:
- Что реализовано
- Какие файлы созданы / изменены
- Что нужно проверить вручную в браузере / админке

---

## Шаг 6: Тестирование

Запусти `npm run build` из директории parent темы (`../PARENT_SLUG/`).
Если сборка упала — остановись и сообщи об ошибке.

Проверь изменённые PHP-файлы:
- Нет `$_POST`/`$_GET` без sanitize
- Нет `echo` без `esc_*`
- Нет `file_put_contents()` / debug-кода
- Новые строки обёрнуты в `__()`

---

## Шаг 7: Коммит + Changelog

1. Добавь запись в `CHANGELOG.md` в секцию `Unreleased`:
   ```markdown
   ### Added
   - [описание реализованного функционала]
   ```

2. Создай коммит:
   - Тип: `feat`
   - Формат: `feat: краткое описание`
   - Включи все изменённые файлы и `CHANGELOG.md`
