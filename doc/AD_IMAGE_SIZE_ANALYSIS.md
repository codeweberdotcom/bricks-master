# Анализ размеров изображений для логотипов клиентов (ad_image size)

## Источник анализа
**URL:** https://sandbox.elemisthemes.com/docs/blocks/clients.html

## Варианты использования логотипов на странице Sandbox

### 1. Swiper Slider (Carousel)
```html
<div class="swiper-container clients mb-0" 
     data-items-xxl="7" 
     data-items-xl="6" 
     data-items-lg="5" 
     data-items-md="4" 
     data-items-xs="2">
  <div class="swiper-slide px-5">
    <img src="./assets/img/brands/c1.png" alt="" />
  </div>
</div>
```
- **Padding:** `px-5` (1.25rem = 20px с каждой стороны)
- **Количество в ряду:** 2-7 в зависимости от экрана
- **Особенности:** Логотипы в слайдере с горизонтальным padding

### 2. Grid Layout (2-4 колонки)
```html
<div class="row row-cols-2 row-cols-md-4 gx-0 gx-md-8 gx-xl-12 gy-12">
  <div class="col">
    <figure class="px-3 px-md-0 px-xxl-2">
      <img src="./assets/img/brands/z1.png" alt="" />
    </figure>
  </div>
</div>
```
- **Padding:** `px-3` (0.75rem = 12px) на мобильных, `px-xxl-2` (0.5rem = 8px) на больших экранах
- **Количество в ряду:** 2 на мобильных, 4 на десктопе
- **Особенности:** Логотипы в grid с адаптивным padding

### 3. Grid с карточками (2-5 колонок)
```html
<div class="row row-cols-2 row-cols-md-3 row-cols-xl-5 gx-lg-6 gy-6">
  <div class="col">
    <div class="card shadow-lg h-100">
      <div class="card-body align-items-center d-flex px-3 py-6 p-md-8">
        <figure class="px-md-3 px-xl-0 px-xxl-3 mb-0">
          <img src="./assets/img/brands/z1.png" alt="" />
        </figure>
      </div>
    </div>
  </div>
</div>
```
- **Padding:** `px-md-3 px-xl-0 px-xxl-3` (адаптивный)
- **Количество в ряду:** 2-5 в зависимости от экрана
- **Особенности:** Логотипы в карточках с тенью

## Анализ размеров изображений

### Рекомендации из веб-стандартов
- **Оптимальный размер:** 200x200 пикселей
- **Для Retina дисплеев:** 300x300 или 400x400 пикселей
- **Формат:** PNG или SVG (для прозрачного фона)
- **Размер файла:** до 1 МБ

### Расчет на основе использования

#### Вариант 1: Swiper (7 логотипов в ряду на XXL)
- Ширина контейнера: ~1400px (XXL breakpoint)
- Padding контейнера: ~40px с каждой стороны
- Доступная ширина: ~1320px
- Padding слайда: 20px × 2 = 40px
- Ширина одного слайда: ~1320px / 7 ≈ 188px
- **Рекомендуемый размер: 200x200px** (с учетом Retina: 400x400px)

#### Вариант 2: Grid (4 колонки на десктопе)
- Ширина контейнера: ~1200px (XL breakpoint)
- Gap между колонками: 12px × 3 = 36px
- Padding: 8px × 2 = 16px
- Доступная ширина: ~1148px
- Ширина одной колонки: ~1148px / 4 ≈ 287px
- **Рекомендуемый размер: 300x300px** (с учетом Retina: 600x600px)

#### Вариант 3: Grid с карточками (5 колонок на XL)
- Ширина контейнера: ~1200px
- Gap: 24px × 4 = 96px
- Padding карточки: 24px × 2 = 48px
- Доступная ширина: ~1056px
- Ширина одной колонки: ~1056px / 5 ≈ 211px
- **Рекомендуемый размер: 250x250px** (с учетом Retina: 500x500px)

## Рекомендации для создания ad_image size

### Вариант A: Универсальный размер (рекомендуется)
```php
add_image_size('codeweber_ad_image', 300, 300, false);
```
- **Ширина:** 300px
- **Высота:** 300px
- **Crop:** false (сохраняет пропорции, не обрезает)
- **Обоснование:** 
  - Подходит для всех вариантов использования
  - Достаточно для Retina дисплеев (2x = 600px)
  - Универсальный размер для логотипов

### Вариант B: Два размера (оптимально)
```php
// Для обычных логотипов
add_image_size('codeweber_ad_image', 200, 200, false);

// Для больших логотипов (в карточках)
add_image_size('codeweber_ad_image_large', 400, 400, false);
```
- **Обоснование:** 
  - Разные варианты использования требуют разных размеров
  - Экономия трафика для маленьких логотипов
  - Качество для больших отображений

### Вариант C: Один размер с обрезкой (не рекомендуется)
```php
add_image_size('codeweber_ad_image', 300, 300, true);
```
- **Crop:** true (обрезка по центру)
- **Не рекомендуется:** Логотипы могут быть обрезаны неправильно

## Итоговая рекомендация

### Рекомендуемый размер: `codeweber_ad_image`
```php
add_image_size('codeweber_ad_image', 300, 300, false);
```

**Параметры:**
- **Название:** `codeweber_ad_image`
- **Ширина:** 300px
- **Высота:** 300px
- **Crop:** `false` (сохраняет пропорции)
- **Использование:** Логотипы клиентов, рекламные изображения, бренды

**Преимущества:**
1. ✅ Универсальный размер для всех вариантов использования
2. ✅ Подходит для Retina дисплеев (2x масштабирование)
3. ✅ Не обрезает логотипы (crop: false)
4. ✅ Оптимальный баланс между качеством и размером файла
5. ✅ Соответствует стандартам веб-дизайна

**Альтернатива (если нужен меньший размер):**
```php
add_image_size('codeweber_ad_image', 200, 200, false);
```
- Для экономии трафика
- Для быстрой загрузки
- Для мобильных устройств

## Примеры использования в коде

### В шаблонах
```php
<?php 
$logo_id = get_post_meta($post_id, 'client_logo', true);
if ($logo_id) {
    echo wp_get_attachment_image($logo_id, 'codeweber_ad_image', false, [
        'class' => 'img-fluid',
        'alt' => get_the_title($post_id)
    ]);
}
?>
```

### В Swiper
```php
<div class="swiper-slide px-5">
    <?php echo wp_get_attachment_image($logo_id, 'codeweber_ad_image'); ?>
</div>
```

### В Grid
```php
<figure class="px-3 px-md-0 px-xxl-2">
    <?php echo wp_get_attachment_image($logo_id, 'codeweber_ad_image'); ?>
</figure>
```

## Сравнение с существующими размерами

| Размер | Ширина | Высота | Crop | Использование |
|--------|--------|--------|------|---------------|
| `codeweber_staff` | 400px | 400px | true | Фото сотрудников |
| `codeweber_ad_image` (рекомендуется) | 300px | 300px | false | Логотипы клиентов |
| `codeweber_ad_image` (альтернатива) | 200px | 200px | false | Логотипы клиентов (компакт) |

## Примечания

1. **Прозрачный фон:** Рекомендуется использовать PNG формат для логотипов с прозрачным фоном
2. **SVG:** Для векторных логотипов лучше использовать SVG (не требует image size)
3. **Retina:** Размер 300x300px обеспечивает качество на Retina дисплеях (2x = 600px эффективно)
4. **Адаптивность:** CSS автоматически масштабирует изображения через `img-fluid` класс
5. **Оптимизация:** Рекомендуется использовать плагины оптимизации изображений (WebP, сжатие)
