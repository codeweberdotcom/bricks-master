# План миграции шаблонов WooCommerce (тема codeweber)

## Цель
Привести переопределённые шаблоны WooCommerce в теме в соответствие с плагином, чтобы убрать предупреждения об устаревших шаблонах и сохранить совместимость.

---

## Этап 0. Подготовка (перед любыми правками)

- [ ] Сделать полный бэкап темы (папка `wp-content/themes/codeweber`).
- [ ] Сделать бэкап БД (на случай отката).
- [ ] Убедиться, что версия WooCommerce в проекте актуальна (обновить плагин при необходимости).
- [ ] Зафиксировать текущие переопределения: список файлов в `themes/codeweber/woocommerce/` и, при необходимости, скриншоты/описание кастомного вида писем и «Моего аккаунта».

## Этап 2. Решение по шаблонам писем (emails)

Сначала принять стратегию: обновлять или убрать переопределения.

### 2.1. Выбор стратегии

**Вариант A — не трогали оформление писем:**

- [ ] Проверить: в теме не менялись шапка/подвал/стили/тексты писем.
- [ ] Если да — отметить «Принята стратегия: удалить переопределения email-шаблонов» и перейти к этапу 3 (удаление).

**Вариант B — меняли оформление писем:**

- [ ] Записать, какие именно файлы менялись (например: email-header, email-footer, email-styles).
- [ ] Отметить «Принята стратегия: обновить только изменённые email-шаблоны» и перейти к этапу 4.

---

## Этап 3. Удаление переопределений email (если выбрана стратегия «удалить»)

Удалять только те файлы, которые не содержат кастомных правок.

- [ ] Удалить из темы папку `woocommerce/emails/` (или только те файлы, что не кастомизировались).
- [ ] Удалить из темы папку `woocommerce/emails/plain/` (аналогично).
- [ ] Удалить из темы папку `woocommerce/emails/block/` (аналогично).
- [ ] В админке WooCommerce → Статус → Шаблоны проверить: предупреждений по удалённым шаблонам быть не должно (будут использоваться из плагина).
- [ ] Отправить тестовые письма (заказ, сброс пароля и т.д.) и проверить отображение.

## Этап 5. Финальная проверка

- [ ] WooCommerce → Статус → Шаблоны: нет предупреждений об устаревших шаблонах.
- [ ] Личный кабинет: вход, дашборд, заказы, просмотр заказа, редактирование аккаунта, адреса, смена пароля.
- [ ] Письма: тестовый заказ, «Новый аккаунт», «Сброс пароля», при необходимости — другие типы писем.
- [ ] Мобильная вёрстка и отображение писем в почтовых клиентах (по необходимости).
