# Шорткод [pdn_sections]

## Описание

Шорткод `pdn_sections` выводит разделы персональных данных для страниц политики конфиденциальности. Используется для динамической генерации таблиц с информацией об обработке персональных данных согласно требованиям законодательства РФ (152-ФЗ «О персональных данных»).

Данные берутся из настроек Redux Framework и выводятся в структурированном табличном формате.

---

## Исходный код

**Файл:** [`functions/admin/admin_privacy.php`](functions/admin/admin_privacy.php:2035)

```php
/**
 * Шорткод [pdn_sections]
 * 
 * Выводит определённые разделы персональных данных согласно заданным индексам.
 * 
 * Атрибуты:
 * - indexes (string) — список индексов разделов, разделённых запятыми. Например: indexes="1,2,3"
 *   Если не указан, будет передан пустой массив.
 * 
 * Внутри функция преобразует строку с индексами в массив целых чисел и вызывает функцию
 * render_personal_data_sections, передавая глобальную переменную $opt_name и массив индексов.
 * 
 * Использование в контенте:
 * [pdn_sections indexes="1,3,5"]
 * 
 * @param array $atts Атрибуты шорткода.
 * @return string HTML с содержимым разделов персональных данных.
 */
function shortcode_pdn_sections($atts)
{
	global $opt_name;

	$atts = shortcode_atts(array(
		'indexes' => '',
	), $atts, 'pdn_sections');

	$indexes = array_map('intval', explode(',', $atts['indexes']));

	return render_personal_data_sections($opt_name, $indexes);
}
add_shortcode('pdn_sections', 'shortcode_pdn_sections');
```

---

## Параметры (Атрибуты)

| Атрибут | Тип | По умолчанию | Описание |
|---------|-----|--------------|----------|
| `indexes` | string | `''` (пусто) | Список индексов разделов через запятую. Например: `indexes="1,2,3"` |

---

## Использование

### Базовое использование (вывод всех разделов)

```php
[pdn_sections]
```

### Вывод конкретных разделов по индексам

```php
[pdn_sections indexes="1,3,5"]
```

### Пример в WordPress контенте

```html
<!-- Вывод разделов 1, 2 и 3 -->
[pdn_sections indexes="1,2,3"]
```

---

## Структура вывода

Шорткод генерирует HTML-таблицу со следующими полями:

| Поле | Описание |
|------|----------|
| Категория субъектов персональных данных | Тип субъектов (физические лица, юридические лица и т.д.) |
| Перечень обрабатываемых персональных данных | Список данных (ФИО, email, телефон и т.д.) |
| Категория обрабатываемых персональных данных | Тип данных (специальные, биометрические, общедоступные) |
| Способ обработки персональных данных | Метод обработки (автоматизированная, неавтоматизированная) |
| Срок обработки персональных данных | Период хранения |
| Способ уничтожения | Метод удаления данных |

### Пример HTML-вывода

```html
<div class="pdn-section">
	<p><strong>7.2.1. Цель обработки "Заполнение формы обратной связи"</strong></p>
	<div class="table-responsive">
		<table class=" table table-bordered">
			<tbody>
				<tr>
					<td style="width:50%; vertical-align: top;"><strong>Категория субъектов персональных данных:</strong></td>
					<td style="width:50%; vertical-align: top;">Физические лица</td>
				</tr>
				<tr>
					<td style="width:50%; vertical-align: top;"><strong>Перечень обрабатываемых персональных данных:</strong></td>
					<td style="width:50%; vertical-align: top;">ФИО, Email, Телефон</td>
				</tr>
				<!-- ... остальные поля ... -->
			</tbody>
		</table>
	</div>
</div>
```

## Источник данных

Данные берутся из настроек Redux Framework с ключами:

- `title_pdn_data{index}` — заголовок/цель обработки
- `cat_sub_pdn_data{index}` — категория субъектов
- `list_pdn_data{index}` — перечень данных
- `cat_pdn_data{index}` — категория данных
- `method_pdn_data{index}` — способ обработки
- `period_pdn_data{index}` — срок обработки
- `destruction_pdn_data{index}` — способ уничтожения

Индексы начинаются с 721 и увеличиваются последовательно.

---

## Зависимости

- Глобальная переменная `$opt_name` — имя опции Redux Framework
- Функция `render_personal_data_sections()` — рендерит секции персональных данных
- Класс `Redux` — для получения опций из Redux Framework

---

## Использование в юридических документах

Шорткод используется в шаблоне политики конфиденциальности для автоматического вывода списка обрабатываемых персональных данных:

```php
// В functions/admin/admin_privacy.php
$content = '...
[pdn_sections]
...';
```

## Bootstrap классы

Вывод использует Bootstrap 5 классы для стилизации:

- `.table` — базовый класс таблицы
- `.table-bordered` — таблица с границами
- `.table-responsive` — адаптивная обёртка

---

## Примечания

1. **Баг в коде:** Функция `render_personal_data_sections()` принимает только один параметр `$opt_name`, но шорткод передаёт два параметра. Это может привести к некорректной работе при попытке фильтрации по индексам.

2. **Глобальная переменная:** Шорткод зависит от глобальной переменной `$opt_name`, которая должна быть определена в области видимости.

3. **Индексы:** При отсутствии параметра `indexes` выводятся все доступные разделы, начиная с индекса 721.

---

## Пример полного использования

```php
// В редакторе Gutenberg или классическом редакторе
[pdn_sections indexes="1,2,3,4,5"]
```

Выведет пять разделов персональных данных в табличном формате.
